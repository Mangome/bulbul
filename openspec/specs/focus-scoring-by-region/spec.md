## Requirements

### Requirement: 基于检测框的合焦评分

当检测到鸟的框后，系统 SHALL 仅在检测框内（相对坐标）计算 Laplacian 二阶梯度方差，而不是全画面评分。评分流程同全画面：下采样（长边 512px）→ Laplacian 卷积 → 分块方差 → Top-3 中位数 → 映射到 1-5 星。

#### Scenario: 检测成功的区域评分

- **WHEN** 检测到鸟框，坐标相对值 (0.2, 0.1, 0.8, 0.9)
- **THEN** 系统在该框内计算 Laplacian 方差，使用现有的映射（50/200/600/1200）转换为评分

#### Scenario: 浅景深背景被正确评价

- **WHEN** 图片为浅景深摄影，背景完全失焦但框架内（鸟身）清晰
- **THEN** 评分基于鸟的清晰度，背景模糊不影响评分

#### Scenario: 复杂背景不影响评分

- **WHEN** 图片背景为高纹理树叶/砖墙，鸟身稍微失焦
- **THEN** 评分基于鸟身实际清晰度，背景纹理高频不拉高评分

### Requirement: 检测失败的降级行为

当检测失败（无框返回）时，系统 SHALL 不给出合焦评分，而是标记为 `focus_score_method = Undetected`。

#### Scenario: 检测失败标记

- **WHEN** 调用检测函数返回空框数组
- **THEN** 设置 `focus_score` = None，`focus_score_method` = Undetected，不计算全画面评分

#### Scenario: 用户收到明确信息

- **WHEN** 查看评分信息
- **THEN** 前端展示"未检测到主体"标记，而不是给出虚假的 1-5 星评分

### Requirement: 多鸟场景的处理

当检测到多只鸟时，系统 SHALL 取置信度最高的框进行合焦评分，但所有框都存储在 `detection_bboxes` 数组供前端使用。

#### Scenario: 多鸟时取最高置信度

- **WHEN** 检测到 3 只鸟，置信度分别为 0.95、0.80、0.70
- **THEN** 仅对置信度 0.95 的框计算合焦评分，其余框存储但不评分

#### Scenario: 所有框都可视化

- **WHEN** 前端 hover 评分组件
- **THEN** 绘制所有 3 个框，标记主评分的框（最高置信度）

### Requirement: 方差阈值映射（暂未重新标定）

区域评分继承现有的方差映射（variance >= 1200 → 5星, >= 600 → 4星, >= 200 → 3星, >= 50 → 2星, < 50 → 1星），后续根据实际照片调整。

#### Scenario: 阈值映射继承

- **WHEN** 计算出的 Laplacian 方差为 800（在 bbox 内）
- **THEN** 映射为 4 星（使用现有的 >= 600 规则）

#### Scenario: 后续重新标定

- **WHEN** 使用 500+ 实际照片测试，发现分布不同
- **THEN** 重新计算阈值，更新映射表（预计 1-2 周）
