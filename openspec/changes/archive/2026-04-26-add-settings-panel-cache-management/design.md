## Context

Bulbul 是一个 Tauri 2 桌面应用（React + Rust），用于 RAW 图片批量相似度分组和导出。当前磁盘缓存（medium/ + thumbnail/ 目录）无大小限制和清理机制，长时间使用后持续增长。同时，应用设置分散在 TopNavBar 的多个 popover 和按钮中，缺乏统一入口。

当前架构：
- **Rust 缓存层**：`cache.rs` 提供 `ensure_cache_dirs()`、`is_cached()`、`write_medium()`、`write_thumbnail()`，无查询大小或删除功能
- **前端缓存层**：`ImageLRUCache`（内存 LRU，400MB 上限）+ `ImageLoader` 封装加载逻辑
- **TopNavBar**：分组参数 popover、省份选择器 popover、检测框按钮、主题切换按钮——4 个独立交互点
- **缓存路径**：Windows `C:\Users\{user}\AppData\Local\bulbul\`，macOS `~/Library/Caches/com.bulbul.app/bulbul/`

## Goals / Non-Goals

**Goals:**
- 提供统一的设置面板，整合分散的配置项
- 让用户感知缓存磁盘占用并支持一键清理
- 清理缓存后自动重新处理当前目录，保持工作流连续性
- 简化 TopNavBar，减少视觉复杂度

**Non-Goals:**
- 不实现按类型（medium/thumbnail）分别清理
- 不实现缓存大小上限自动清理
- 不实现孤立缓存文件检测和清理
- 不重新设计整体设置架构（仅添加面板 + 迁移现有控件）

## Decisions

### D1: 设置面板形式——右侧滑出抽屉

**选择**：右侧固定宽度（320px）滑出面板，带半透明遮罩

**替代方案**：
- 模态对话框：遮挡画布，无法同时查看图片
- 侧边栏（常驻）：占用画布空间

**理由**：滑出抽屉在打开时不影响画布浏览，与 TopNavBar 的浮动面板风格一致，关闭后完全不占空间。

### D2: 分组参数从 TopNavBar popover 完全迁移到设置面板

**选择**：移除 TopNavBar 的 IconTune 按钮，参数仅在设置面板中调节

**替代方案**：保留 TopNavBar 快捷入口，同时设置面板中也有参数控制（双入口）

**理由**：双入口会导致状态同步复杂性（两处修改同一数据），且分组参数不属于高频操作，放在设置面板中更符合语义。TopNavBar 已有 6+ 个按钮，精简更优。

### D3: 检测框开关迁移到设置面板

**选择**：移除 TopNavBar 检测框按钮，开关仅在设置面板中

**替代方案**：保留在 TopNavBar

**理由**：检测框是低频开关，与设置面板"外观"区域语义吻合。TopNavBar 精简后视觉更清爽。

### D4: 缓存清理后自动重处理

**选择**：清理成功后，如果 `currentFolder` 存在，自动调用 `startProcessing(currentFolder)`

**替代方案**：仅清缓存，提示用户手动重新打开目录

**理由**：用户清理缓存通常是为了释放空间后继续使用，自动重处理减少操作步骤。`process_folder` 会因缓存未命中而重新生成缩略图。

### D5: 清理确认使用内联确认而非模态对话框

**选择**：在设置面板缓存区域内，点击「清理缓存」后按钮变为「确认清理」红色按钮（二次点击确认）

**替代方案**：弹出独立确认模态框

**理由**：减少 UI 复杂度，内联确认模式简单直观，无需额外组件。

### D6: Rust 缓存函数使用 tokio::fs 异步遍历

**选择**：`get_cache_size()` 和 `clear_all_cache()` 使用 `tokio::fs::read_dir` 异步遍历

**替代方案**：使用 `std::fs` 同步遍历 + `spawn_blocking`

**理由**：缓存文件数量通常在千级，`tokio::fs::read_dir` 足够高效，代码更简洁。避免 `spawn_blocking` 的线程池开销。

### D7: 主题切换保留在 TopNavBar

**选择**：主题切换按钮保留在 TopNavBar，不迁移到设置面板

**理由**：主题切换是高频操作（用户经常在亮暗模式间切换），保留在顶栏方便快速访问。

## Risks / Trade-offs

- **[缓存清理耗时]** → 大量文件删除时可能阻塞。缓解：异步遍历 + 删除，前端显示 loading 状态。实测千级文件删除在 1 秒内完成。
- **[重处理期间无法操作]** → 清理后自动 `startProcessing` 会弹出进度对话框，用户需等待。缓解：这是预期行为，且缓存未命中时处理速度较快（仅生成缩略图，跳过 pHash 计算）。
- **[分组参数可发现性降低]** → 从顶栏按钮迁移到设置面板后，新用户可能找不到。缓解：设置按钮使用通用齿轮图标，点击后参数区域在面板顶部。
