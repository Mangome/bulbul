## ADDED Requirements

### Requirement: 手动检查 GitHub Release 更新
系统 SHALL 允许用户在应用内手动检查 GitHub Releases 提供的更新元数据，并仅比较当前平台可安装的新版本。

#### Scenario: 发现可安装的新版本
- **WHEN** 用户在设置面板点击“检查更新”，且更新端点返回高于当前版本、适配当前平台的新版本
- **THEN** 系统 SHALL 显示最新版本号、更新说明，并提供“下载并安装”动作

#### Scenario: 当前已是最新版本
- **WHEN** 用户在设置面板点击“检查更新”，且更新端点未返回比当前版本更高的适配版本
- **THEN** 系统 SHALL 明确告知用户当前已是最新版本

### Requirement: 下载并安装更新
当存在可安装更新时，系统 SHALL 在用户显式确认后下载签名安装包、执行安装，并在安装完成后重启应用或提示重启以完成更新。

#### Scenario: 用户安装可用更新
- **WHEN** 用户点击“下载并安装”，且下载与签名校验成功
- **THEN** 系统 SHALL 安装该更新并完成应用重启流程

#### Scenario: 安装过程中显示进行状态
- **WHEN** 系统正在下载或安装更新
- **THEN** 系统 SHALL 显示进行中的状态，并阻止重复触发新的安装请求

### Requirement: 更新失败反馈
系统 MUST 在检查、下载、签名校验或安装失败时向用户提供可见的失败反馈，并保留应用当前版本可继续使用。

#### Scenario: 检查更新失败
- **WHEN** 更新端点不可达或返回无效元数据
- **THEN** 系统 SHALL 显示检查失败状态，且不影响用户继续使用当前应用版本

#### Scenario: 安装更新失败
- **WHEN** 下载、签名校验或安装过程发生错误
- **THEN** 系统 SHALL 显示安装失败信息，且不得将应用置于不可交互状态
